Web Storages

HTML5 Web storage allows us to store data in local machine. New HTML5 API offering advance feature than traditional cookies. Web storage is more secure and large amount of data can store locally, without affecting website performance. Many browser allows 4 KB per cookies, but we can store like 5 MB of data in web storage with sending to server. Date stored in web storage will be key value pair. The key and the value are always string.

JavaScript can easily access the web storage which increment the best performance for the website more than traditional server-side and relation database. There are two types of web storage object available

  • sessionStorage
  • localStorage

Difference between localStorage and sessionStorage

Local storage is similar to session storage but without expiring time, but session storage data will be cleared whenever the tab is closed or browser is closed. date stored in website either local storage or session storage based the protocol of the web page.

sessionStorage

sessionStorage only available within the browser tab or window. Whenever the tab is closed the session storage data will be deleted.

localStorage

This allows to store data locally without expiring time. Data will be available in the browser even after the window or browser is closed. localStorage kept even between storage which means data stored in edge can not be access using chrome.

HTML5 Web Storage Object

HTML5 provides two object to store data on the client

  • window.localStorage - store data with no expiring date / time.
  • window.sessionStorage - store date with expiring time.

Web storage objects

  • localStorage - display local storage object
  • localStorage.clear() - to remove everything in local storage
  • localStorage.setItem() - to store date in local storage, it takes key and value parameter
  • localStorage.getItem() - to display data stored in the local storage, it takes key as a parameter
  • localStorage.removeItem() - to remove stored item from the local storage, it takes key as a parameter
  • localStorage.key() - to display data store in a localStorage, it takes index a parameter
